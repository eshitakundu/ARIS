The System

Fixed DAG. Seven parallel branches. One personalised verdict.

Same input, same path, every time. A branch that fails lowers confidence, not the score — missing data is honest, not hidden. Click any pipeline stage to expand.

Decompose
Resolve tool, detect registry, validate queries.
An LLM agent resolves the tool name, detects the package registry, normalises the GitHub slug, and runs a validate_queries skill before any branch fires.

03 →
Score
PythonExec computes every dimension deterministically.
The custom PythonExec node runs deterministic Python inline. All six dimension scores use documented, named-constant formulas — no LLM touches a number. Repo verdict overrides the band (`ALREADY_USED` → `KEEP`).
Python only

LLM receives computed scores, structured findings, and the repo profile — writes rationale, alternatives, caveats, and a per-dimension narrative that cites your specific stack. It cannot alter any figure.

Stack Compatibility
Your repo: manifests + README + tree + languages → Python + LLM
Parses requirements.txt, pyproject.toml, package.json, Pipfile, and setup.cfg. Detects your repo's archetype, identifies competing tools already declared, and produces a personalised engineer commentary displayed in the brief as a pull-quote. Verdict labels per-repo: ALREADY_USED · MIGRATION_REQUIRED · FIT · POOR_FIT.

Inside your brief

What lands in your inbox.

Overall, no adoption work is needed — just keep an eye on version compatibility as the ecosystem evolves."
